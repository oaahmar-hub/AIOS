Executive summary

The last five days turned AIOS from looks live into actually answers a customer honestly. Three things moved:

  1. The real brain is reconnected. The live WhatsApp reply path runs through the Omar personality engine, quotes real inventory from the Truth Bridge, and is under a hard no-fabrication rule — no invented units, prices, sizes, or links.
  2. The delivery loop is closed. The hosted runtime now actually sends auto-replies, deduplicates them, and never goes silent when the brain fails (graceful fallback).
  3. We can now see failures. /api/health/deep checks every link in the reply chain and reports green/red per component — the silent failures behind every past outage now surface immediately.

Data quality (Truth Bridge Quality) moved 48.3 → 55.5 / 100 with an authorized inventory feed and anchor-recovery work — accounted honestly, with no fabricated links.

01  What shipped this week

1. The brain — reconnected, honest, and knowledgeable

  • Reconnect the real brain (46aa9b1) — wired omar_personality_engine into the live reply path. Replies come from AIOS's own personality layer, not a passthrough.
  • Hard no-fabrication rule (c185b57) — the first live reply invented a listing (800 sqft / 850k that don't exist). The prompt now forbids stating any specific unit, price, size, availability, owner, or link not present in context; the bot says it will check and confirm in Omar's natural style instead.
  • Knowledge connection — quotes REAL inventory (f8483bd) — parses area / building / bedrooms / budget from the message (English and Arabic, incl. Arabic-Indic digits and «غرفة وصالة», ألف/مليون), searches the resolver index, and injects a VERIFIED context block. Only genuinely quotable rows are used; junk areas and misaligned rows filtered; sizes quoted without a unit (source mixes sqm/sqft). 237 quotable rows today. No anchor in the message → no results → the no-fabrication rule keeps the reply at "I'll check."

2. WhatsApp delivery — the loop is closed

  • Send auto-replies (6d144ad) — the hosted runtime now actually delivers replies to customers.
  • Graceful fallback (67d8f54) — a failing brain never means silence.
  • Dedupe (7a201bb) — deduplicated by message_id behind an actionable gate, ending duplicate-message incidents.
  • Wasender webhook signature (1350d06) accepted on the hosted endpoint.

3. Observability — silent failure is over

  • /api/health/deep (48c6a61) — public, read-only, sends nothing to customers. Checks every link the live reply depends on: runtime, resolver_db, webhook_auth, reply_mode, wasender_send, brain_n8n_openai (a real n8n ping that catches a dead OpenAI key instantly), fallback_reply. Returns healthy / degraded / down plus reply_chain_live and a plain-English list of what's broken.

4. Security & data

  • Dashboard no longer public (8cc832b) — stopped serving every .html publicly; the dashboard now requires auth.
  • Authorized Scorpion inventory feed (63565b0) — 140 de-duplicated units with provenance; idempotent content-hash IDs. Building-unit anchors 7 → 17.
  • JVC building-name recovery (f43bef6) & anchor repair (d2a52bc) from corrupted/packed inventory strings; live-portal verdicts fed back into candidate bridges (5eb29d9); Unusable-rows audit documented honestly (da2358b).
  • The Command Deck (e1df4d4) — honest-AI marketing site, bilingual, with a live heartbeat.
