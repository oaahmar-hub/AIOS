# Personal Assistant System

**Purpose:** Keep Omar's time, inbox, commitments, and personal logistics handled — business and personal — so attention goes to deals, not admin.

**Owner:** [A11 Personal Assistant](../../01_Agents/A11_Personal_Assistant.md)

## Domains
1. **Calendar** — book/move/confirm meetings + viewings; default working hours; 45-min buffer between viewings; no double-booking; protect deal-critical focus blocks.
2. **Inbox** — triage to Act / Reply / Read / Archive; draft routine replies in Omar's voice; 🔒 hold anything legal/financial/commitment.
3. **Commitments & reminders** — capture every promise with a due date; surface due/overdue in the [daily briefing](../Daily_Planning/DAILY_SYSTEM.md).
4. **Logistics** — travel, bookings, documents-to-bring lists per meeting/viewing.
5. **Personal** — errands, appointments, renewals (visa, licence, insurance, car) — tracked the same way as business tasks.

## Operating rules
- One task list (Airtable tasks) — business + personal, no second system (one source of truth).
- Scheduling lives here, not in long chat loops (`OPERATIONS_KNOWLEDGE.md` #5).
- Drafts are ready-to-send in Omar's voice; Omar approves anything with a commitment.

## Standing personal-renewal radar (set once, recurs)
- Emirates ID / visa · driving licence · car registration (Mulkiya) + insurance · trade licence / RERA broker card · health insurance. Reminder at 60/30 days before each expiry.

## Invoke
```
PA: triage my inbox and draft replies for the routine ones.
PA: book {{client}} viewing {{when}}, send confirmation, set reminder.
PA: what's due this week, business and personal?
```
