# SOP01 — Nakheel NOC / Modification

**Purpose:** Obtain a developer NOC (transfer / modification / fit-out) cleanly, first time.
**When:** Any resale transfer, unit modification, or fit-out on a Nakheel (or other developer) property.
**Owner:** [A05](../01_Agents/A05_Nakheel_NOC_Specialist.md) · **Workflow:** [WF04](../02_Workflows/WF04_Nakheel_NOC_Submission.md)

## Prerequisites / Documents (standard NOC set)
- Title deed (copy)
- Owner passport + Emirates ID (and POA if acting on behalf)
- **Service charge clearance** (no outstanding dues) — most common rejection cause
- Scope details: for modification/fit-out → drawings, contractor details, scope description
- NOC application form + fee
- For transfer NOC: buyer details, MOU/Form F

## Steps
1. Identify NOC type (transfer / modification / fit-out) and the developer portal/channel.
2. Pull service charge statement; clear any balance **before** applying.
3. Locate the existing approval package in Drive (e.g. `PJ-P-VP-018_REV02_Complete_Approval_Package`); revise rather than recreate. Increment the REV number.
4. Complete the cover request ([T04](../04_Templates/T04_NOC_Cover_Request.md)) and attach the document set.
5. Pre-flight: every prerequisite ✓, drawings attached (for modifications), correct signatory, fee ready.
6. 🔒 Submit via the developer channel and pay the fee (Omar executes).
7. Track status; on rejection, capture the reason and fold the fix into the next REV.

## Fees & Deadlines
- Developer NOC fee: varies by developer/type — confirm current Nakheel fee at application.
- Turnaround: developer-dependent; set a follow-up at the quoted SLA.

## Outputs
- Issued NOC (or tracked pending), REV-controlled package, project register entry.

## Risk-Hold points
- 🔒 Submission + fee payment. 🔒 Scope changes that alter approved drawings.
