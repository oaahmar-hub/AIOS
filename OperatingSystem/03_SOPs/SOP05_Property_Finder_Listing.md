# SOP05 — Property Finder Listing

**Purpose:** Publish an accurate, compliant, high-converting listing.
**When:** New listing or refresh on Property Finder (and other portals).
**Owner:** [A03](../01_Agents/A03_Listings_Manager.md) · **Workflow:** [WF03](../02_Workflows/WF03_PF_Listing_Publish.md)

## Prerequisites / Documents
- Verified unit (via [A12](../01_Agents/A12_Unit_Finder_Operator.md))
- Valid RERA advertising permit ([SOP04](SOP04_RERA_Advertising_Permit.md))
- Photos / floorplan / video from Drive
- Defensible price (comps via [A08](../01_Agents/A08_Research_Feasibility_Analyst.md))

## Steps
1. Verify the unit; if URL-only and unresolved, confirm manually before listing.
2. Confirm permit number on hand (no permit → stop, route to SOP04).
3. Pull comparables; set the price within a defensible band.
4. Write copy from [T01](../04_Templates/T01_Listing_Copy.md): title → highlights → location → terms; only compliant, verifiable claims.
5. Attach media; ensure floorplan + min photo count.
6. Publish with the permit number; set portal + price + date.
7. Log to pipeline tracker; set a 30-day refresh/stale check.

## Fees & Deadlines
- Portal subscription/credits per listing; permit fee per SOP04.
- Refresh stale listings every ~30 days.

## Outputs
- Live listing with permit number, comps on file, refresh reminder.

## Risk-Hold points
- 🔒 No permit → no publish. 🔒 Price outside comps range. 🔒 Unverified unit/authorisation.
