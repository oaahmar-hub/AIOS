# WF07 — Contract Renewal

**Flow:** Expiry radar → decision → notice (90d) → new terms → re-register
**Trigger:** A lease/contract is within 120 days of expiry (radar set at signing).
**Lead agent:** [A06 Contracts & Renewals](../01_Agents/A06_Contracts_Renewals.md) · **Supports:** A04, A02

## Steps
1. **Radar** — reminders fire at **120 / 90 / 60 / 30 days** before expiry.
2. **Decision** — renew as-is, renew with change, or non-renew. If rent change/non-renewal, the **90-day** written notice window governs (`LEGAL_KNOWLEDGE.md` #5).
3. **Rent check** — if increasing, validate against the RERA rental increase calculator via [A04](../01_Agents/A04_Compliance_Officer.md).
4. **Draft the notice/renewal** from [T07](../04_Templates/T07_Renewal_Notice.md); state the legal window + service method (notary/registered mail for eviction).
5. 🔒 **Serve** — Omar serves the notice / signs the renewal.
6. **Re-register** — new tenancy → re-Ejari ([WF06](WF06_Ejari_Registration.md)) within 30 days.
7. **Update** CRM end date + project register; set the next renewal radar.

## Outputs
- Decision recorded, compliant notice/renewal drafted, served (by Omar), re-Ejari triggered, new expiry radar set.

## Done when
- Renewal executed or notice served within the legal window, re-registered, next radar set.

## Hard rule
- Never miss the 90-day window. The radar exists so this never happens by surprise.
