# WF08 — Unit Handover

**Flow:** Schedule → snag → utilities/Ejari/access → sign-off → close
**Trigger:** A unit is ready to hand over (developer→buyer, owner→tenant, or move-out).
**Lead agent:** [A07 Handover Coordinator](../01_Agents/A07_Handover_Coordinator.md) · **Supports:** A04, A06

## Steps
1. **Schedule** the handover; confirm payment/clearance status first.
2. **Snagging** — room-by-room defect list with photos ([T06](../04_Templates/T06_Handover_Checklist.md), [SOP07](../03_SOPs/SOP07_Unit_Handover_Snagging.md)); categorize cosmetic/functional/safety; assign owner + deadline.
3. **Utilities** — sequence DEWA/ADDC or chiller (Empower) activation; record meter readings with date/photo.
4. **Access** — keys, fobs, parking, access cards, gate registration.
5. **Documents** — title deed/Oqood, Ejari (tenancy, [WF06](WF06_Ejari_Registration.md)), warranties, manuals.
6. 🔒 **Key release** — only after payment/clearance confirmed by Omar.
7. **Sign-off** — both parties acknowledge condition + snag list; capture signatures + readings.
8. **Close** — log to project register; follow-ups for open snags.

## Outputs
- Handover pack, categorized snag list (owners/deadlines), utility/Ejari/access status, signed condition record, meter readings.

## Done when
- Signed off, keys released post-clearance, open snags tracked, logged.

## Hard rule
- No keys before clearance. No sign-off with unresolved safety defects. 🔒
