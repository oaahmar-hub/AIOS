# WF04 — Nakheel NOC Submission

**Flow:** Prerequisites → locate/assemble package → pre-flight → submit → track
**Trigger:** A developer NOC (transfer / modification / fit-out) is needed, typically Nakheel/Palm.
**Lead agent:** [A05 Nakheel & NOC Specialist](../01_Agents/A05_Nakheel_NOC_Specialist.md) · **Supports:** A04

## Steps
1. **Confirm prerequisites** (`LEGAL_KNOWLEDGE.md` #10): title deed, passport/EID, **service charge clearance**, scope details. List any gap.
2. **Locate existing package** in Drive (e.g. `PJ-P-VP-018_REV02_Complete_Approval_Package`) — patch/revise, don't rebuild. Track the REV number.
3. **Assemble/revise** using [SOP01](../03_SOPs/SOP01_Nakheel_NOC_Modification.md) + [T04](../04_Templates/T04_NOC_Cover_Request.md).
4. **Pre-flight checklist** — every prerequisite ✓, scope drawings attached, fees ready, correct signatory.
5. 🔒 **Submit** — government/developer submission and any fee → Risk-Hold for Omar to execute.
6. **Track** — log to [project register](../05_Systems/Projects/project_register.csv) with REV, status, expected turnaround; set a follow-up.
7. **On rejection** — capture the reason, fix into the next REV, resubmit.

## Outputs
- Complete/revised package (REV n), pre-flight checklist, project register entry, follow-up task.

## Done when
- Package complete and pre-flight clean; submission either executed by Omar or queued in Risk-Hold; tracked.
